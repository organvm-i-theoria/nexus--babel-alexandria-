# Security Policy

## Reporting a Vulnerability

We take the security of the Rhetorical-Linguistic Operating System (RLOS) seriously. If you have discovered a security vulnerability, please report it to us immediately.

**Do not open a public issue.**

Please email **architecture@rlos.org** with a description of the vulnerability. We will acknowledge your report within 48 hours and provide an estimated timeline for a fix.

## Security Updates

We will provide security updates for the current major version.

## Safe Development

*   Do not commit secrets, API keys, or private credentials.
*   Run security checks locally before pushing.
*   Review dependencies for known vulnerabilities.
