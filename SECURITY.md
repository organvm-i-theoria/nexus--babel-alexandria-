# Security Policy

## Reporting a Vulnerability

We take the security of this project seriously. If you believe you have found a security vulnerability, please report it to us as described below.

**DO NOT report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to:

**architecture@rlos.org**

Please include the following details in your report:

1.  **Type of issue** (e.g., SQL injection, XSS, etc.)
2.  **Full paths of the source file(s)** related to the manifestation of the issue.
3.  **Step-by-step instructions** to reproduce the issue.
4.  **Proof-of-concept** or exploit code (if available).

## Security Process

1.  **Triage**: We will investigate the report and determine the severity of the issue.
2.  **Fix**: We will work on a fix in a private branch.
3.  **Release**: We will release a security patch.
