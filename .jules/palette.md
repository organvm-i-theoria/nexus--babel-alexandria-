## 2024-05-22 - [Standardizing Entry Points]
**Learning:** In a new repository with heavy documentation but no code, the primary "User Experience" is the "Developer Experience" of navigating the files. Long, non-standard filenames act as friction.
**Action:** Always establish a standard `README.md` and clean up key documentation filenames (e.g., `DESIGN.md`) as the first step to make the project accessible to human readers.
