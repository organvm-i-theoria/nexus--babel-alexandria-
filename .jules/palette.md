## 2024-05-23 - Standardizing Repository Entry Points
**Learning:** In early-stage repositories without code, Developer Experience (DX) *is* the UX. Long, descriptive filenames for design docs (e.g., `# Theoria...`) create friction for developers using CLI tools.
**Action:** Always rename foundational documents to standard conventions (`DESIGN.md`, `roadmap.md`) and ensure a clear `README.md` exists to guide new contributors.
