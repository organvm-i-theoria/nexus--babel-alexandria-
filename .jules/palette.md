# Palette's Journal

## 2024-05-22 - [Initializing Repository UX]
**Learning:** In a completely empty repository, the most critical UX is Developer Experience (DX). Standardizing entry points (README.md) and cleaning up filenames (like deeply nested or long design docs) significantly reduces cognitive load for new contributors.
**Action:** Always check for `README.md` and sane file naming conventions as the first step in a new project.
