## 2024-05-23 - [DX is UX too]
**Learning:** In an empty repository, the most "delightful" UX improvement is establishing a clear entry point (README) and basic hygiene (.gitignore). Without these, a developer feels lost and frustrated, which is a poor user experience.
**Action:** Always check for basic repository health (README, .gitignore, clean filenames) before diving into feature code or visual tweaks.
